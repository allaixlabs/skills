# RNR.md — R&R: AI 자동화 vs 인간 개입 경계

모든 권한을 루프에 넘기면 원치 않는 대규모 코드 수정이나 아키텍처 왜곡이 발생한다.
명확한 통제 경계선을 확립해, 경계를 넘는 작업은 자동 진행을 멈추고 사람을 호출한다.

---

## 경계표

| AI 자동 처리 영역 (Autonomous Execution) | 인간 승인·호출 영역 (Human-in-the-Loop) |
|---|---|
| Lint / 포맷팅 / 타입 오류 수정 | **DB 스키마(Schema) 구조 변경** |
| 컴파일·CI 실패 로그 기반 단순 버그 수정 | 데이터 손실 리스크가 있는 마이그레이션 |
| 누락된 단위 테스트 코드 보강·작성 | **인증·인가·결제·보안 관련 로직 수정** |
| 코드 변경점에 따른 문서·주석 업데이트 | 최초 기획 문서(PRD) 범위를 벗어나는 기능 수정 |
| 변수명 변경 등 단순 네이밍 리뷰 반영 | 테스트는 통과하나 아키텍처 설계 의도와 달라지는 큰 변경 |

---

## 가드레일 (Guardrails)

1. **브랜치 제한**: AI가 자동으로 코드를 고쳐 푸시할 수 있는 브랜치를 제한한다.
   - 예: `ai/*`, `bot/fix-*` 브랜치에서만 자동 푸시 허용. `main`/`release/*` 직접 푸시 금지.
2. **인간 호출 알림 (Webhook)**: 인간 승인 기준에 걸리면 자동 수정을 **중단**하고
   Slack 등으로 알림을 보내 멈춘다.
   ```bash
   # 인간 호출 알림 예시 (SLACK_WEBHOOK_URL 은 사용자가 설정)
   curl -X POST "$SLACK_WEBHOOK_URL" \
     -H 'Content-Type: application/json' \
     -d '{"text":"⛔ Human-in-the-Loop 필요: <사유>. 자동 진행을 중단했습니다."}'
   ```
3. **판단 기준**: 변경 diff가 아래 중 하나라도 건드리면 인간 호출.
   - 마이그레이션 파일 / 스키마 정의
   - `auth`, `payment`, `security`, 권한 검사 로직
   - PRD의 Out of Scope 항목
   - 공개 API 계약(인터페이스) 변경

---

## 적용 방식

- 자동 처리 영역은 감시 루프(`WATCH-LOOP.md`)에서 self-healing으로 처리한다.
- 인간 승인 영역은 **절대 자동 커밋하지 않는다.** 변경 제안만 만들고 사용자 확인을 받는다.
